# monte carlo tree search

## markov decision processes

Let's set up a simple model for the world and investigate how one might make optimal decisions. 
Our model will be discrete. Let's represent the set of all possible *states* in our
model as $\mathcal{S}$. For some state $s \in \mathcal{S}$, let $\mathcal{A}(s)$ be the set of all
available *actions* in that state. Taking an action from one state leads us to a new state. 
For $s, s' \in \mathcal{S}$ and $a \in \mathcal{A}(s)$, Let
$\mathcal{T}(s, a, s')$ be the *transition probability* of reaching state $s'$ after taking action $a$ in
state $s$. Let $\mathcal{R}(s, a, s')$ be the *reward* gained after such a transition. For simplicity,
let's assume that every sequence of actions eventually reaches a terminal state (a state $s$ such that
$\mathcal{A}(s)$ is the empty set) and hence there is no way to accumulate infinite reward. We call
the tuple $(\mathcal{S}, \mathcal{A}, \mathcal{T}, \mathcal{R})$ a *Markov Decision Process (MDP)*.

An *optimal decision rule* is a mapping from every state $s \in \mathcal{S}$ to an action in
$\mathcal{A}(s)$, such that follwing the optimal decision rule for every state you visit in a traversal maximizes the expected
cumulative reward of the traversal. For an MDP like ours, this is simply a lookup table and we'll denote it as $\pi^*(s)$.
To us explain our optimal decision rule, we'll define associated functions called *values* and *action values*. 
The value of a state $s$ is the expected cumulative reward from following
our optimal decision rule starting from state $s$. It is denoted by $V^*(s)$. The action value of
an action $a$ in a state $s$ is expected cumulative reward we get from following our optimal 
decision rule starting from state $s$, provided we take action $a$ first. It is denoted by $Q^*(s, a)$. It 
should be relatively clear that

$$
\pi^*(s) = \arg\max_{a \in \mathcal{A}(s)} Q(s, a)
$$

$$
V^*(s) = \max_{a \in \mathcal{A}(s)} Q(s, a)
$$

$$
Q^*(s, a) = \sum_{s' \in \mathcal{S}}\mathcal{T}(s, a, s')(\mathcal{R}(s, a, s') + V^*(s'))
$$

For your curiosity, this is the 
[Bellman equation for MDPs](https://inst.eecs.berkeley.edu/~cs188/textbook/mdp/solve.html#421-the-bellman-equation).

If we can compute $Q^*(s, a)$ for every possible pair $(s, a)$ of state and action in our MDP,
then we know the optimal decision rule. However, even from a glance, it should be clear that computing these
action values is very hard. To know $Q^*$, we must find $V^*$. But to find $V^*$, we must know
$Q^*$! If your interested in one way to solve this problem, I encourage you to look at the
[Value Iteration Algorithm](https://inst.eecs.berkeley.edu/~cs188/textbook/mdp/value-iteration.html).
However, value iteration requires you to loop over every single tuple $(s, a, s')$ of state,
action, and resulting state in the entire MDP many times. For larger MDPs, the computation time is
massive. Furthermore, you must wait for the algorithm to finish entirely before reasonably concluding even a 
single optimal decicion rule. This is because value iteration solves the MDP entirely, finding every
optimal decision rule simultaneously.

## what is monte carlo tree search?

*Monte Carlo tree search* is an algorithm for finding the optimal decision rule for a given starting state
$s$. To make the computation tractable, the algorithm takes two major shortcuts:

1.  The algorithm only computes the action values for states and actions which you can actually reach
    from the starting state $s$, hence why the algorithm is formally known as a *tree search*. For many
    MDPs, this cuts down the number of values we need to compute immensely.
1.  The algorithm uses random sampling to estimate exptectations of cumulative reward, hence why this
    algorithm is formally knows as a *Monte Carlo* tree search. In general, Monte-Carlo methods use
    random sampling to estimate complex expectations which are intractable to solve exactly.

To explain, let's suppose that we know the optimal decision rule for our MDP. Then the Monte Carlo
way to compute $Q^*(s, a)$ would be to start at state $s$ and take action $a$. Then, keep taking
actions accoring to the optimal decision rule until termination, keeping track of the total accumulated reward.
Repeat this process of running a full simulation hundreds, or thousands, or millions of times. The 
average total accumulated reward over all simulations converges to the true value of $Q^*(s, a)$.

But we don't know the optimal decision rule! How can we randomly sample from a distribution that we
don't actually know yet? We need to devise a new sampling strategy which may start off wrong, but
eventually corrects it's course and converges to the correct one. The correct way to sample is: 
"at every step choose the action which grants us the highest total expected cumulative reward". The sampling
method that we will construct will be, informally, "choose the action based on a strategy which gets
us as close to the highest total expected cumulative reward without knowing which action grants us this
upfront". As the number of trials increase, this strategy will eventually always pick the optmial action,
and hence we still converge to the correct value. Most importanly, this strategy minimizes the error
on its way to convergence. We borrow this strategy from a famous problem in reinforcement learning.

## The multi-armed bandit problem

Suppose that you have a large number of coins and you enter a room with multiple slot machines. You know nothing
about the expected ayout of the slot machines. Some slot machines could have higher expected payouts than other slot
machines. How do you maximize your expected total payout? Well, we wan't to play the slot machine with the
highest expected payout as mant times as possible!

At a given step, we can compute the empirical average of the payout each slot machine observed so far.
A reasonable strategy would be to choose the slot machine which the highest average payout, right?
But if we have only played a certain slot machine a small number of times, then the average payout
might be far from the expected payout. Lets devise a better metric than the average. At a given step, set some probability threshold
$p$, which should be small. Then, for each slot machine, compute
an estimate for the expected payout of said slot machine such that the probability of the *true* 
expected payout being higher than the estimate is at most $p$. Therefore if we reject a slot machine based
on this estimate, the probability of falsely rejecting that slot machine is at most $p$.

We can construct such an estimate using *Hoeffding's Inequality*. Let $X_1, X_2, \ldots, X_n$ be a
sample of $n$ payouts from a given slot machine. Let $\overline{X}$ be thier average and $\mathbb{E}[X]$
be the true expected payout. Finally, assume that each payout is bounded in the range $[a, b]$. Then

$$
\mathbb{P}(\mathbb{E}[X] > \overline{X} + t) \leq \exp\left(-\frac{2nt^2}{(b - a)^2}\right)
$$

If we set the right hand side equal to $p$ and solve for $t$ we get

$$
t = (b - a)\sqrt{\frac{-\ln(p)}{2n}}
$$

and now we have the estimate we are looking for. 

Finally, we need a way to set the value of $p$.
Recall that $p$ is the bound on the probabilty of falsly rejecting a slot machine. This should relate to the number
of tokens we have spent. If we have only spent a few tokens, then the total uncertainty is high and
it is okay if we falsely reject a slot machine to make room for one which has proved more lucrative so far.
However, if we have spent a large number of tokens, we want to be absolutely sure that we have found 
the true optimal slot machine. A paper called
[Finite-time Analysis of the Multiarmed Bandit Problem](https://homes.di.unimi.it/~cesabian/Pubblicazioni/ml-02.pdf)
proved that setting $p = N^{-4}$, where $N$ is the number of tokens spend so far, is best, and that
the resuting estimate

$$
UCB_i = \overline{X}_i + (b - a)\sqrt{\frac{2\ln(N)}{n_i}}
$$

called the *Upper Confidence Bound (UCB)*, yields the optimal algorithm for maximizing profit.

## the monte carlo tree search algorithm

To find the optimal action from a starting state $s$, Monte Carlo Tree Search treats every state in 
the MDP as its own independent multi-armed bandit problem. The algorithm iteratively loops through 
two main steps to build a localized decision tree:

1.  **Sampling:** Starting from $s$, we traverse a continuous path to a terminal state by always 
choosing the action that maximizes the UCB estimate. Crucially, if an action has never been tried 
($n_i=0$), its UCB estimate is mathematically infinite. Therefore, we will *always* pick an 
unexplored action over an explored one. Once we select an unexplored action, we have no empirical 
data to compute UCB estimates for the subsequent states, so we simply complete the rest of the 
trajectory to the terminal state by picking actions entirely at random.
2.  **Updating:** Once the terminal state is reached and the total accumulated reward is calculated, 
we walk back up the exact path we just sampled. For every state-action pair we visited, we increment 
its visit count $n_i$ and update its empirical average $\overline{X}_i$ using the new reward.

We repeat this loop until our computational budget is exhausted. Finally, we look at the actions 
available from our root state $s$ and select the one with the highest empirical average (or the 
highest visit count) as our optimal move.
